🤝 谈判对手匹配系统 · 使用说明
欢迎使用谈判对手匹配系统！本系统可帮助买卖双方一对一随机配对，适合模拟谈判、课堂练习或团队活动。
🌐 访问地址
•	用户匹配页面：https://match.containers.snapdeploy.dev/
•	管理后台页面：https://match.containers.snapdeploy.dev/admin
________________________________________
👤 用户操作指南（参与者）
1. 打开网页
在电脑或手机的浏览器中打开活动组织方提供的链接（建议使用 Chrome、Edge 或 Safari 浏览器）。实测微信就可以
2. 填写信息
进入页面后，你将看到以下表单：
•	选择身份：请根据你被分配的角色，选择 “我是买方” 或 “我是卖方”。
•	你的昵称：请填写便于对方识别的名字（如“市场部-张三”）。
•	补充信息（选填）：可填写职位、部门等备注，匹配成功后对方会看到。
  
3. 开始匹配
点击蓝色按钮 “开始匹配”，系统将自动为你寻找对手。
•	若当前无对应身份的等待者，页面会显示 “等待卖家/买家加入...”，请保持页面打开，不要关闭或刷新。
•	当有对手进入时，双方页面将自动更新，展示对方的昵称和补充信息。
4. 匹配成功
匹配成功后，请根据活动规则主动联系对手开始谈判（联系方式由组织方统一通知）。
  电脑开三个浏览器+一个手机端测试 匹配，快
⚠️ 注意事项
•	请勿刷新或关闭页面，否则会退出匹配队列。
•	每个设备仅能选择一种身份，请勿重复提交。
•	若长时间卡在等待状态，请及时联系活动管理员。

🛠️ 管理员操作指南（组织者）
1. 进入管理后台
•	在浏览器中打开管理页面：https://match.containers.snapdeploy.dev/admin
2. 设置总人数
•	在输入框中填入总参与人数（必须为偶数，系统将自动平分买卖双方名额）。
•	点击 “应用” 按钮保存设置。
•	默认总人数为 200（买卖双方各 100 人）。
哪一半是买家哪一半是买家可以提前抽个签，应该有很多小程序什么的，只需输入人数
3. 实时状态监控
管理后台会动态显示：实测有点不实时hh
•	总人数上限
•	已加入的买方 / 卖方人数
•	当前等待匹配的买方 / 卖方人数
点击 “刷新状态” 可获取最新数据。
4. 活动前准备
•	提前设置好总人数。
•	建议先安排少量人员测试匹配流程是否顺畅。
•	若一方名额已满，可适当调高总人数（需在参与者加入前调整）。
